# Build instructions for AMO reviewers — ChatGPT-Voyager (GPT-Voyager) v1.6.9 (Firefox)

This document tells a Mozilla add-on reviewer exactly how to reproduce the
submitted `.xpi` from the source code in this archive.

Submitted artefact: **`chatgpt-voyager-1.6.9-firefox.xpi`**
Source repository: <https://github.com/TanChuping/chatgpt-voyager>

---

## 1. Operating system

Any of:

- Linux (Ubuntu 22.04+ / Debian 12+ / Fedora 38+)
- macOS 12+
- Windows 10+ (PowerShell 5 or 7)

The build is platform-independent because zipping is done in-process via
`jszip` (no native `zip` / `Compress-Archive` invocations). Tested on
Windows 11 with Node 24.12.

## 2. Required toolchain

| Tool        | Version              | Install                                                  |
| ----------- | -------------------- | -------------------------------------------------------- |
| **Node.js** | >= 18 (tested 24.12) | <https://nodejs.org/>                                    |
| **Bun**     | >= 1.0               | Linux/macOS: `curl -fsSL https://bun.sh/install \| bash` |
|             |                      | Windows: `powershell -c "irm bun.sh/install.ps1 \| iex"` |

Plain `npm` works too — replace `bun` with `npm` in every command below.
Both `bun.lock` and `package-lock.json` are shipped, so dependency
resolution is reproducible either way.

No global packages are required. All build dependencies are declared in
`package.json` and pulled into a local `node_modules/` by the install step.

## 3. Step-by-step build

```bash
# 1. Unzip the source archive, then cd into it.
cd chatgpt-voyager-1.6.9-source

# 2. Install dependencies.
bun install               # or: npm install

# 3. Build + package the Firefox .xpi.
bun run pack:firefox      # or: npm run pack:firefox
```

That single `pack:firefox` command runs:

1. `vite build --config vite.config.firefox.ts` → produces `dist_firefox/`
2. `node scripts/pack-firefox.cjs` →
   - copies `dist_firefox/` to a stage directory,
   - strips any `key` field from `manifest.json` (only used for local
     unpacked-extension development to keep the Chrome ID stable across
     reloads — Firefox ignores it but AMO rejects it),
   - zips the staged tree into
     `firefox_release/chatgpt-voyager-1.6.9-firefox.xpi` using `jszip`
     (pure-JS, forward-slash entry names — produces a spec-conformant
     archive on every OS).

The resulting `firefox_release/chatgpt-voyager-1.6.9-firefox.xpi` matches
the submitted artefact (zip metadata timestamps may differ but file
contents are byte-equivalent).

## 4. What the bundler does

Vite (with `@crxjs/vite-plugin`) takes the TypeScript + React + Tailwind v4
sources under `src/` and produces ES-module JS bundles minified by esbuild.
**No code obfuscation.** Source-to-bundle mapping is straightforward:

| Source                          | Bundle role            |
| ------------------------------- | ---------------------- |
| `src/pages/content/index.tsx`   | content script         |
| `src/pages/background/index.ts` | background script      |
| `src/pages/popup/`              | toolbar popup UI       |
| `src/pages/options/`            | options page           |
| `src/pages/pageWorld/`          | injected MAIN-world JS |
| `public/`                       | static assets verbatim |
| `src/locales/`                  | i18n message bundles   |

## 5. Firefox-specific notes

- `vite.config.firefox.ts` aliases `mermaid` to **mermaid-legacy@9.2.2**
  because Firefox's parser chokes on mermaid v11's ESM output. The
  Chrome build uses mermaid v11 directly.
- `browser_specific_settings.gecko.id = "gpt-voyager@local"` is set in
  the Firefox vite config (overrides the chrome manifest's value at
  build time).
- `background.scripts` (an array) is used instead of `service_worker`
  for Firefox MV3 compatibility — generated by the firefox vite config.
- `optional_host_permissions` in `manifest.json` (`https://*/*` and
  `http://*/*`) are used only for a user-opt-in feature that lets the
  extension act on non-ChatGPT pages; they are _optional_ and never
  granted by default.

## 6. Privacy / safety claims

- **No remote code execution.** All scripts ship inside the bundle.
- **No analytics.** No telemetry, no auto-update beacons, no third-party
  trackers.
- **No external network calls** beyond ChatGPT itself (the host the
  extension augments). The conversation-export feature reads from
  ChatGPT's own API via the user's existing browser session.
- All user data (timeline pins, folders, prompts, favourites) lives in
  `chrome.storage.sync` / `localStorage` — the extension never sends it
  anywhere.
